# 自驾路书地图 MVP 需求说明

## 1. 产品定位

一个仅供个人使用的静态自驾路书网站。地图是主要界面，用户通过日期和海拔筛选目的地，点击地图标记后查看地点详情。

第一版在本地运行，后续可直接部署为静态网站，不包含账号、后台和网页编辑功能。

## 2. 核心体验

### 地图

- 首页以全屏地图为主。
- 支持路网图层与卫星图层切换。
- 默认根据整段行程自动调整地图视野。
- 地图范围主要位于中国大陆。

### 目的地标记

- 地图只展示目的地，不展示住宿、补给等其他类型的点。
- 标记使用清爽、克制的现代旅行杂志风格。
- 标记显示 `Day 1`；同一天存在多个目的地时显示 `Day 1 · 1`、`Day 1 · 2`。
- 点击标记后，桌面端展开左侧详情栏，并将地图平移、放大至该地点。
- 移动端以底部抽屉代替左侧栏。

### 行驶路线

- 理想方案是根据公路生成实际行驶路线。
- MVP 在没有地图路线规划 API 的情况下，按 Markdown 中的目的地顺序使用简洁线段连接。
- 界面明确注明该路线为示意连线，并非实际导航路线。
- 后续接入高德等路线规划服务时，应保留静态降级方案。

### 筛选

- 提供日期区间双端滑块，可选择起始日和结束日。
- 筛选后隐藏区间外的目的地及对应路线。
- 测试版提供固定的“海拔”筛选项，使用海拔范围双端滑块。
- 日期和海拔筛选同时生效。

### 地点详情

详情栏展示：

- Day、日期及当日序号
- 地名
- 经纬度与海拔
- 当日里程与预计驾驶时间
- 地点简介
- 注意事项
- 固定标签
- 外部链接：导航、住宿、攻略、文章、视频或其他
- 图片集：首图大图、其余图片横向缩略图，点击后全屏浏览

图片同时支持项目内相对路径和网络 URL。

### 行程概览

展示：

- 行程名称与副标题
- 封面图
- 起止日期
- 总天数
- 总里程
- 简短引言

## 3. 内容管理

- 一段行程对应一个 Markdown 文件。
- Markdown 使用 YAML Front Matter 保存结构化行程与地点数据。
- Markdown 正文用于较自由的行程介绍和补充说明。
- 第一版直接编辑文件，不提供网页编辑后台。

## 4. 视觉方向

- 现代旅行杂志风格。
- 以地图、摄影和留白为主，避免传统导航软件的工具感。
- 使用克制的中性色，搭配一个来自公路、落日或地貌的强调色。
- 标题可使用具有编辑感的字体，正文保持高可读性。
- 动效应轻量，重点用于详情栏、地图定位和图片浏览。

## 5. 响应式要求

- 优先适配桌面浏览器。
- 桌面端：左侧详情栏 + 地图。
- 移动端：全屏地图 + 底部详情抽屉。
- 日期和海拔筛选在窄屏下折叠为筛选面板。

## 6. MVP 验收标准

1. 网站能读取一份符合模板的 Markdown 路书。
2. 地图能显示全部目的地和 Day 标记。
3. 同一天的多个目的地能正确编号并按顺序连接。
4. 点击标记能打开对应详情，并定位地图。
5. 日期与海拔范围筛选能正确更新地点和路线。
6. 路网与卫星图层可以切换。
7. 本地图片和网络图片均能展示。
8. 桌面端与移动端均可正常使用。

## 7. 暂不包含

- 登录、多人使用和权限管理
- 网页内编辑路书
- 云端数据库
- 实时导航和实时路况
- 住宿、补给、景点等其他地点类型
- 未配置 API Key 时的公路自动路线规划
